# Как перенести контекст диалога Cursor на другой компьютер

## Варианты переноса контекста

### 1. Автоматический перенос через аккаунт Cursor (рекомендуется)

Cursor синхронизирует некоторые данные через ваш аккаунт:
- Настройки редактора
- Расширения (частично)
- **НО:** История чатов хранится локально и не синхронизируется автоматически

### 2. Ручной перенос истории чатов

История чатов Cursor хранится локально. Для переноса:

#### Windows:
```
%APPDATA%\Cursor\User\workspaceStorage\
```

Или полный путь:
```
C:\Users\<ВашеИмя>\AppData\Roaming\Cursor\User\workspaceStorage\
```

#### macOS:
```
~/Library/Application Support/Cursor/User/workspaceStorage/
```

#### Linux:
```
~/.config/Cursor/User/workspaceStorage/
```

**Шаги:**
1. На старом компьютере: Найдите папку `workspaceStorage` для вашего проекта
2. Скопируйте всю папку проекта из `workspaceStorage`
3. На новом компьютере: Откройте проект в Cursor
4. Скопируйте сохраненную папку в соответствующее место на новом компьютере

**Примечание:** Идентификатор папки в `workspaceStorage` основан на пути к проекту, поэтому может отличаться на разных компьютерах.

### 3. Использование файла PROJECT_CONTEXT.md (рекомендуется)

Создан файл `PROJECT_CONTEXT.md` в корне проекта, который содержит:
- Описание всех модулей системы
- Структуру проекта
- Архитектуру
- Важную информацию о БД
- Инструкции по установке

**Преимущества:**
- ✅ Файл в репозитории - доступен всем разработчикам
- ✅ Версионируется через Git
- ✅ Не зависит от локальных настроек Cursor
- ✅ Можно обновлять по мере развития проекта

### 4. Экспорт через Cursor Chat (если доступно)

В некоторых версиях Cursor есть возможность экспорта истории чата:
1. Откройте панель чата
2. Найдите опцию экспорта (если доступна)
3. Сохраните историю в файл
4. Импортируйте на новом компьютере

### 5. Ручное сохранение важных диалогов

Если есть важные диалоги, которые нужно сохранить:
1. Скопируйте текст диалога из Cursor
2. Сохраните в файл (например, `CHAT_HISTORY.md`)
3. Добавьте в проект или сохраните отдельно

## Рекомендуемый подход

**Для переноса контекста проекта:**

1. ✅ Используйте `PROJECT_CONTEXT.md` - он содержит всю важную информацию о проекте
2. ✅ Обновляйте `PROJECT_CONTEXT.md` при значительных изменениях
3. ✅ Используйте Git для версионирования контекста
4. ✅ При необходимости скопируйте папку `workspaceStorage` вручную

**Для продолжения конкретного диалога:**

1. Экспортируйте важные части диалога в `PROJECT_CONTEXT.md`
2. Или сохраните в отдельный файл `CHAT_HISTORY.md`
3. На новом компьютере откройте файл и используйте его как контекст для нового диалога

## Быстрый старт на новом компьютере

1. Клонируйте/скопируйте проект
2. Прочитайте `PROJECT_CONTEXT.md` для понимания структуры
3. Настройте конфигурацию (`config/config.php`, `config/database.php`)
4. Запустите миграции БД
5. Откройте проект в Cursor
6. При необходимости скопируйте `workspaceStorage` для истории чатов

---

**Совет:** Регулярно обновляйте `PROJECT_CONTEXT.md` с важной информацией из диалогов, чтобы не терять контекст при переходе между компьютерами.
